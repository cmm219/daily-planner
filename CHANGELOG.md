# Changelog

All notable changes to this project are documented here.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to a four-segment monotonic version scheme: `major.minor.patch.micro`.

Recommended early-alpha starting version: `0.1.0.0`. Reserve `1.0.0.0` for the first stable/public release.

Bump types:
- `major` — breaking product/platform change, incompatible data contract change, major relaunch, or stable milestone
- `minor` — meaningful feature/capability added, such as a new surface, integration, workflow, or major UX direction
- `patch` — bug fix or behavior correction
- `micro` — very small shipped change, such as copy tweak, docs-only, cleanup, styling polish, or small non-risky internal adjustment

## [0.2.0.0] - 2026-06-04

### Added
- Terminal-style themeable settings panel with left subtabs: Colors, Headers / Bold, Fonts, Sections, Presets. Opens via the gear icon or Ctrl+,.
- Visual color picker popup (SV square + hue slider + live preview + editable hex) so users pick colors by eye instead of typing hex codes.
- Functional title-bar traffic-light dots: red closes the settings dialog, green toggles dialog size, yellow minimizes.
- Font family + size theming and customizable bold headers; all settings persist to the planner JSON.
- Sections subtab: rename any blue section header, hide/show sections (checkbox), add custom sections, and reset names to default — all live-previewed and persisted.
- Checkbox app icon (black background, blue check) replacing Flet's default, generated by `scripts/make_icon.py`.
- Headless QA suites: `tests/test_settings_model.py`, `tests/test_settings_integration.py`, `tests/test_planner_actions.py` (63 tests) driving every button, keyboard shortcut, the settings dialog, and the section overrides against a stub Page.

### Fixed
- Fonts subtab crash: `ft.Dropdown` requires `on_select` (not `on_change`) in Flet 0.80; the family dropdown would have thrown on load.
- Fonts subtab spacing: the custom-font field's floating label no longer overlaps the family dropdown.
